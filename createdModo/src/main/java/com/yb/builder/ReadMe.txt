一、建造者的使用场景：
主要是参数过多，参数又存在可选与不可选问题
1.全部放到构造函数中，参数过多，影响阅读性。并且参数容易传错，存在隐藏的bug。
2.解决1的问题，构造函数中只传必填参数，其余通过set来补充。
    如果必填的参数过多，又存在1的问题。
    如果参数之间有联系，需要联合校验，比如var1<var2等，通过set不能完成
    如果对象创建后不能更改，就不能使用set
3.要解决12的问题，可以通过建造者模式。
二、工厂模式和建造者模式的区别
工厂模式负责创建一类对象，根据对象的类型来决定创建其中某一个。
建造者模式负责创建一个对象，主要解决创建时参数过多的问题。
